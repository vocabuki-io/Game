# 企画書：ターン制プリズンブレイク対戦ゲーム（仮題）

> ステータス：初版ドラフト（v0.1）。数値は暫定・要プレイテスト。
> 対象：スマホ専用ブラウザ（PWA）／基盤：Cloudflare。
> 元アイデア：`docs/idea-dump.md`。設計判断の骨格は専門家対談＋ユーザー4決定で確定済み。

---

## 1. コンセプト / ピラー

**一言ピッチ**
> 「模範囚を演じながら脱獄を企てる囚人」と「証拠を掴んで捕まえたい看守」の、1対1・約10分の心理戦カードバトル。

**3本のピラー**
1. **手軽（ゴッフィ路線）** — ルールは一目で分かり、手札を見ればその場でやることが決まる。1マッチ〜10分。
2. **読み合い（主役体験）** — 看守は力押しできない。「違反の現行犯」を押さえない限り囚人を倒せない。囚人は演技で欺く。
3. **逆転性** — 手札運・非対称の霧・追跡フェーズにより、最後の1手まで勝者が分からない。

**確定した骨格（ユーザー決定）**
- 1v1（囚人1・看守1）を主軸。多人数・非対称は将来拡張。
- リアルタイム同室（Cloudflare Durable Objects + WebSocket）。
- 主役は「読み合い・頭脳戦」。サブの勝ち筋として「武器を集めて看守を倒す」も搭載。
- 1マッチ目標〜10分。

---

## 2. プレイ概要

- **プレイヤー**：囚人（P）1人 vs 看守（G）1人。
- **1マッチ**：暫定 **全14日**（＝14ラウンド）。実時間で約10分になるようフェーズを軽量化。
- **盤面**：すごろく風のノードグラフ（マス）。囚人が移動・工作し、看守が巡回・監視する。
- **手札**：毎ラウンド、山札からランダムに配られる**行動カード**で行動する（手札運）。
- **情報**：非対称。囚人には未探索の「未公開マス（霧）」があり、看守は囚人が作った隠しマス以外を見渡せる。
- **決着**：下記いずれかの勝利条件。期日（14日目終了）到達で看守勝ち。

---

## 3. 勝利条件

### 囚人（P）の勝ち
- **(a) 脱獄達成**：複数ある脱獄方法（§8）のいずれかを完遂して盤外（門/トンネル出口）へ脱出。
- **(b) 武器で撃破**：武器パーツ（§9）を集めて組み立て、看守を倒す（サブの勝ち筋）。

### 看守（G）の勝ち
- **(a) 期日到達**：14日目終了まで脱獄も撃破もされず監督しきる → 期日に処刑。
- **(b) 現行犯→捕縛/撃破**：囚人の**違反を現行犯で確認**して一発アウトにし、続く**追跡フェーズ**（§5）で捕縛または撃破。
  - ⚠️ **模範囚（＝違反未確認の囚人）は倒せない**。これがゲームの核。

> 設計意図：看守は「疑わしい」だけでは何もできず、確定した証拠（現行犯）が要る。
> 囚人は表向き模範囚を演じ、裏で脱獄/武器を進める。両者の情報戦が主役体験を作る。

---

## 4. ターン進行（1日 = 1ラウンド）

全14日。1日は次の4フェーズ。テンポ優先で各フェーズは短く。

| フェーズ | 内容 |
|---|---|
| ① 点呼（イベント抽選） | その日のイベント（§10）を1枚めくる。点呼マスへの集合が要求される日もある。 |
| ② 行動 | P・Gが手札から行動カードを選び提出（同時提出＝読み合い）。移動・工作・監視・検査など。 |
| ③ 判定 | 提出された行動を解決。移動・工作進捗・接触・検査結果を反映。**現行犯の成立判定**もここ。 |
| ④ 夜（違反チェック/点呼） | 規定（§6）の遵守を確認。未帰房・禁制品露見などがあれば**一発アウト**→追跡フェーズへ。 |

- 現行犯が成立しなければ、そのまま翌日へ。期日（14日目④）到達で看守(a)勝ち。

---

## 5. アウト判定 ＋ 追跡フェーズ（核 / メーターは無し）

### アウト判定（一発アウト）
- **疑いメーターは存在しない。** 疑いは蓄積・可視化しない。
- 看守が**違反を現行犯で確認した瞬間**（例：禁制品を検査でヒット、点呼に不在、立入禁止区でトンネル工作を目撃）に、その囚人は**即アウト**。
- 逆に言えば、**確定情報が揃うまで看守は手を出せない** → 模範囚は倒せない。

### 追跡フェーズ（アウト発生後の山場）
アウトが出ると通常進行を中断し、追跡フェーズに移行。1v1のクライマックス。

- **囚人の選択肢**
  - **逃走**：全力で出口（門/トンネル）へ。追跡を振り切って脱出できれば囚人(a)勝ち。
  - **刃向かう（反撃）**：武器/道具で看守に抵抗。倒せれば囚人(b)勝ち、失敗すれば撃破/捕縛。
- **看守の手段**
  - **出入口の封鎖**：門・扉を閉じて脱出ルートを断つ。
  - **逃げ道潰し**：分岐マスを塞ぎ、囚人の選べる進路を狭める。
  - **マスを被せて捕縛/撃破**：囚人のいる/進むマスを制圧して捕らえる。
- **決着**：捕縛/撃破されれば看守(b)勝ち。逃げ切り or 反撃成功なら囚人勝ち。

> 追跡フェーズは「読み合い（どこへ逃げる/どこを塞ぐ）」＋短いアクション性で、逆転の見せ場を作る。
> 暫定：追跡は数手（3〜5手）で決着する軽量ミニゲームとして設計。

## 6. 規定（ルール）と違反

看守が「現行犯」を取る対象となる、囚人に課された規定。既定例（暫定）：

| 規定 | 違反の露見条件（例） |
|---|---|
| 点呼応答 | 点呼フェーズに指定マス（点呼マス）に不在 → 夜に発覚 |
| 指定エリア在室 | 立入禁止区（門付近・工房裏など）で看守に目撃/隣接される |
| 禁制品非所持 | 手荷物検査イベントで禁制品（工具・武器パーツ・鍵）がヒット |
| 隠し工作の非露見 | 囚人が作ったマス（トンネル等）を看守が発見 |

- **模範囚**＝上記いずれの違反も現行犯確認されていない状態。看守は倒せない。
- 囚人は「バレる前に隠す・演じる・逃げる」の管理が戦略の柱。

## 7. 行動カード

- 毎ラウンド、山札からランダムに手札を配布（ゴッフィ的な手札運）。
- P/G それぞれ専用の山札を持つ（役割で行動が異なるため）。
- 既定カテゴリ：

| カテゴリ | 例（囚人） | 例（看守） |
|---|---|---|
| 移動 | 歩く／急ぐ（複数マス） | 巡回移動 |
| 工作 | トンネルを掘る／鍵をいじる（脱獄準備を進める） | ― |
| 隠蔽 | 禁制品を隠す／アリバイ工作（演技） | ― |
| 情報 | 周囲を偵察（未公開マスを開く） | 監視／目撃を狙う |
| 妨害 | ― | 検査／封鎖／点呼招集 |
| 作業 | 刑務作業でリソース獲得 | ― |

- カードは基本1枚1行動。強力なカードは出現率低め/リスク付きで運と戦略を両立。

## 8. 脱獄方法（複数ルート）

囚人(a)勝ちの達成手段。複数用意して「毎回違うプラン」を可能に。各ルートは"準備ゲージ"を段階的に進める。

1. **トンネル工作** — 独房/工房で複数ラウンド掘り進める。囚人が作った隠しマスとして生成（看守に不可視）。ただし検査/工事イベントで露見リスク。
2. **面会経由の変装脱出** — 面会イベントで外部から変装具を受け取り、面会室ルートで脱出。タイミングが命。
3. **賄賂で鍵入手** — 作業でリソースを貯め、鍵を入手 → 門から脱出。禁制品（鍵）所持中は検査に弱い。

> MVPでは**トンネル工作1種＋（時間があれば）鍵入手**を実装し、残りは章として設計だけ確定。

---

## 9. 武器パーツ（サブの勝ち筋）

- 囚人(b)勝ちの手段。**断片（パーツ）を複数集めて組み立て → 看守を撃破**。
- 入手：作業・イベント・特定マスから。
- **高リスク設計**：パーツは禁制品扱い。手荷物検査でヒットすれば即アウト → 追跡へ。
- バランス：脱獄より能動的で爽快だが、露見リスクが高く一発勝負になりやすい。読み合い主役を壊さない位置づけ。

---

## 10. イベント

点呼フェーズで抽選。盤面・リソース・露見に影響する共有イベント。

| イベント | 効果（暫定） |
|---|---|
| 手荷物検査 | 指定/全囚人の所持品チェック。禁制品ヒットで現行犯→アウト。囚人最大のリスク。 |
| 工事 | マップが変化（通路封鎖/開通）。トンネル進捗に影響（有利/露見）。ランダム性の源。 |
| 刑務作業 | 参加でリソース獲得。ただし作業中は特定マスに拘束され動けない。 |
| 面会 | 外部からアイテム/変装具を受け取れる。変装脱出ルートの起点。 |

---

## 11. 2Dマップ

- **形式**：すごろく風ノードグラフ（マスと接続）。移動はカードで規定歩数/隣接移動。
- **既定ノード**：独房・中庭・工房・食堂・面会室・門（出口）。
- **重要方針**：**マスは今後ランダム性を強化**する（ランダム生成・ランダムイベントマス）。
  - よって固定レイアウトに依存しない**データ駆動のノード定義**にする（ノード種別・接続・属性をデータで持つ）。
  - 囚人が作った隠しマス（トンネル等）は動的に追加され、看守には不可視。

---

## 12. 情報の非対称性（非対称の霧）— 核の一部

- **囚人視点**：未探索の場所は **「未公開マス（霧）」**。偵察/移動で開く。囚人は盤面全体を最初は把握できない。
- **看守視点**：盤面をほぼ全て見渡せる。**ただし囚人が作ったマス（トンネル等の隠し工作）だけは見えない。**
- **共通**：囚人の手札・脱獄進捗・所持禁制品は看守に非公開。看守は**現行犯の確認でのみ**アウトを取れる。

> 非対称の霧が「囚人は隠す・欺く／看守は探して確定させる」という主役体験を成立させる。

---

## 13. 運と戦略のバランス

- **3すくみ**：手札運 × 位置取り（マップ/霧） × "現行犯を避ける/押さえる"読み合い。
- 手札運で毎回展開が変わりつつ、隠蔽・偵察・封鎖の判断で実力差が出る。
- マスのランダム性強化と噛み合わせ、「同じ盤面の暗記ゲー」になるのを防ぐ。
- 数値（山札構成・カード効果・脱獄ゲージ長・追跡手数）は暫定 → プレイテストで調整。

---

## 14. MVPスコープ

**入れる**
- 1v1・リアルタイム同室。
- 全14日のターン進行（4フェーズ）。
- **一発アウト＋追跡フェーズ**。
- **非対称の霧**（囚人=未公開マス／看守=囚人の作ったマスのみ不可視）。
- 脱獄1種（トンネル工作）＋武器サブ勝ち筋（簡易）。
- イベント4種、ノード約10、行動カード最小セット。

**入れない（将来拡張）**
- 多人数・非対称モード。
- 非同期モード。
- 脱獄ルートのフルバリエーション、アート/サウンド、マネタイズ。

## 15. 技術付録（Cloudflare）

| 役割 | 機能 |
|---|---|
| フロント配信（スマホPWA） | Cloudflare Pages / Workers Assets |
| ゲーム進行・API・マッチング | Workers |
| 1部屋の状態＋リアルタイム同期 | **Durable Objects + WebSocket**（1ルーム=1 DO） |
| 戦績/プレイヤー | D1（SQLite）or KV |
| カード素材・画像 | R2 |
| bot対策（任意） | Turnstile |

- **権威サーバ型**：ゲーム状態と判定はDOが権威を持つ。手札・霧・脱獄進捗はサーバ側に保持し、
  各クライアントには**その役割が見てよい情報だけ**を送る（＝非対称の霧とチート耐性を同時に担保）。
- **フロント**：縦持ち・タッチ最適化。PWA化でホーム追加＆オフラインシェル。
- **IP配慮**：ゴッドフィールドの名称・カード・画像は流用せず、**オリジナルのカードセット/世界観**で作る。

## 16. 将来拡張

- **多人数・非対称モード**：囚人複数 vs 看守1〜2。人狼的な「誰が本気で脱獄狙いか」の面白さ。バランス難度は上がる。
- **非同期モード**：各自の都合で1手ずつ＋通知。スマホ手軽さ重視の別モード。
- **マップ生成の強化**：ランダム生成・イベントマスの拡充（§11の方針の延長）。

## 17. サンプル対戦トレース（ルール整合性の自己検証）

1v1、トンネル脱獄を狙う囚人P vs 看守G。抜粋。

- **1日目** ①点呼（イベント：刑務作業）。②P=作業でリソース獲得／G=中庭を巡回。③解決。④違反なし。
- **3日目** ①点呼（イベント：なし）。②P=独房でトンネル工作（隠しマス生成、Gには不可視）／G=食堂を監視。③工作進捗+1。④違反なし＝**Pは模範囚**、Gは手を出せない。
- **7日目** ①点呼（イベント：**手荷物検査**）。②P=禁制品(工具)を隠すカードで隠蔽を試みる／G=検査を実行。③判定：隠蔽成功なら露見せず、失敗なら**工具ヒット=現行犯→アウト**。ここが読み合いの山。
  - 例：隠蔽失敗 → **アウト → 追跡フェーズ**。P=逃走を選び門へ／G=門を封鎖し分岐を潰す → Pがトンネル出口（自作の隠しマス、Gに不可視）へ回り込めれば**囚人(a)脱獄勝ち**、読まれて塞がれれば**看守(b)勝ち**。
- **14日目** 誰も脱獄/撃破せず期日到達 → ④で**看守(a)勝ち**。

> 検証結果：各勝ち筋（脱獄・武器・看守期日・看守現行犯）がすべて到達可能で、
> 「模範囚は倒せない」核（＝現行犯が無い限り看守は無力）が一貫して機能することを確認。
> 支配戦略が無いよう、隠蔽/検査/霧の数値はプレイテストで詰める。
